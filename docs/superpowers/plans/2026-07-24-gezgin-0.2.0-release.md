# Gezgin 0.2.0 Dependency Rollup and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run all Dependabot checks, consolidate the twelve updates compatible with the current KMP/KSP publication toolchain into one verified Gezgin `0.2.0` release, and publish all four artifacts through the existing tag-driven Maven Central workflow.

**Architecture:** Apply overlapping version-catalog and GitHub Actions changes together on `codex/gezgin-0.2.0-release` so dependency families and CodeQL action versions cannot drift. Keep runtime/API behavior unchanged, verify the combined dependency graph locally and in CI, merge one release PR, then tag the exact merge commit.

**Tech Stack:** Kotlin Multiplatform, Gradle 8.14, Kotlin 2.4.10, KSP 2.3.10, Compose Multiplatform 1.11.1, GitHub Actions, Maven Central.

## Global Constraints

- Preserve the public API dumps and require `apiCheck` to stay green.
- Keep AndroidX Lifecycle at `2.10.0`; `2.11.0` requires the separate AGP 9.1+/Gradle 9.3.1+/compileSdk 37 Android-KMP plugin migration.
- Keep Kotlin at `2.3.21`; KSP `2.3.10` targets Kotlin language level 2.3 and emits incompatible-metadata diagnostics for Kotlin `2.4.10`.
- Keep CodeQL `init` and `analyze` pinned to the same `v4.37.3` commit.
- Keep all GitHub Actions pinned to full 40-character commit SHAs.
- Publish `gezgin-core`, `gezgin-processor`, `gezgin-mvi`, and `gezgin-test` with `VERSION_NAME=0.2.0`.
- Do not create `v0.2.0` until the release PR is merged with all required checks green.
- Do not create the GitHub Release until Maven Central smoke verification succeeds.
